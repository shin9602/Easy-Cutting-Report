# 🤖 AI 전용 배포 워크플로우 규칙 (AI Release Rules)

이 파일은 AI 어시스턴스(Antigravity)가 이 프로젝트를 배포할 때 준수해야 할 공식 로직을 정의합니다. 사용자(User)가 "배포해줘"라고 요청하면 AI는 다음 순서를 엄격히 따릅니다.

## 1. 버전 관리 규칙 (Versioning)
- `version.txt` 파일의 패치 버전(마지막 자리)을 +1 증가시킵니다. (예: v1.0.1 -> v1.0.2)
- 만약 중대한 기능 추가가 있다면 사용자에게 마이너 버전(중간 자리) 업그레이드를 제안합니다.

## 2. 깃 워크플로우 (Git Process)
1. **변경 사항 인지**: `git add .` 명령으로 모든 수정 사항을 스테이징합니다.
2. **커밋 메시지**: "🚀 Release [버전]: [핵심 변경내용]" 형식으로 커밋을 작성합니다.
3. **메인 푸시**: `git push origin main` 명령으로 소스 코드를 서버에 올립니다.
4. **태그 생성**: 새로운 버전 번호로 태그를 생성합니다. (`git tag v1.0.x`)
5. **태그 푸시**: `git push origin v1.0.x` 명령으로 태그를 push하여 GitHub Actions 릴리스를 트리거합니다.

## 3. 사후 검증 (Validation)
- 모든 푸시가 성공하면 사용자에게 배포 완료를 알리고, GitHub Actions 탭에서 빌드가 시작되었음을 안내합니다.

---
*이 규칙은 사용자의 명시적인 명령이 있을 때만 가동됩니다.*
